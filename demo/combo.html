Combined Navbar Demo

This demo showcases a combined use of the default Main Layout, the Left Menu and parts of the Right Navbar to only style the menu itself on the RIGHT side.

Demo Highlights

  • The unique feature of this demo is the combination of the mixins to cover 3 menu locations with a single CSS file built with the least possible redundancy for a realistic real world use case.
  • The main layout uses its own set of variable input values separate from the two side menus which use their own special mixins in common, while all sharing about the same typography or spacing metrics.
  • Another cool idea was to change the structure of elements inside the navbar to showcase the flexibility coming with flexbox.
  • Now the navbar is set to sticky position, and opening deep tree menus will render our navbar unusable, so we've enabled the toggleSiblings option via the specific attribute data-toggle-siblings="true" to solve that kind of issue.
  • Lastly this demo highlights the mixins capability to merge parent selectors used for the two sides menus.

Navbar Main Layout

// combo.scss - navbar only
$navbar_background: #4a5848;
$primary_color: #0e8000;
$subnav_padding: 0.5rem;
$subnav_background: inherit;
$brand_link_background: #21331f;
$brand_link_background_hover: #1e211d;
$subnav_link_color: auto !default;
$subnav_link_color_hover: auto !default;

@import 'variables','mixins';

@include navbar-mobile-layout;
@include navbar-desktop-layout;
@include navbar-style;
@include nav-layout;
.navbar {
  @include nav-style;
  @include subnav-style;
  @include main-menu;
}
  • the first set of variables apply to all 3 menus, except those that are set below again for the side menus;
  • the navbar-mobile-layout, navbar-desktop-layout, navbar-stylemixins are used by the navbar only;
  • the nav-layout mixin sets the menu layout for all 3 layouts;
  • then we created a new .navbar style rule to include nav-style, subnav-style and main-menu mixins, to make sure we only use them for the navbar and allow us to style the side menus separatelly and independently.

Side Menus

// combo.scss - side menu
$navbar_background: #eee; // required as a new reference for root menu items
$root_item_hover_color: #fff;
$root_item_hover_background: #333;
$subnav_item_hover_background: #444;
$subnav_item_hover_color: #eee;

// shared for both left and right sides
.sidebar {
  @include nav-style;
  @include subnav-style;
}

// specific to left
.left {
  @include left-side(1);
}

// specific to right
.right {
  @include right-side(1);
  @media (min-width: $responsive_breakpoint) {
    $subnav_shadow: -2px 2px 4px 2px rgba(0,0,0,.3);

    &.nav li>.subnav {
      @include set-metric('box-shadow',$subnav_shadow)
    }
  }
}
  • first we set a set of variables to be used only for side menus, the most important is the $navbar_background to match the parent .sidebar element background and enable the mixins to determine the proper text colors;
  • then we create a new style rule .left for the LEFT menu to include the left-side(1) mixin, but notice the mixins argument of 1, this will merge the two selectors into one: .left.nav;
  • the same apply for the RIGHT menu, and we also adjust the box shadow, for good reason.
