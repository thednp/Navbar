Bootstrap Extended Navbar

Unlike the other Bootstrap demo this one showcases how to easily integrate Navbar with its full features and markup with Bootstrap. Only a couple of variables are required with your special flavor and you're ready to go.

Demo Highlights

  • The build showcases how to recreate the other demo just by using the default Navbar markup and adjusting some variables. It might strike you that you can simply replace the original Bootstrap 4/5 Navbar JavaScript plugin and its CSS sources completely, since Navbar doesn't require Dropdown and Collapse components (and their dependencies) in order to work.
  • The idea of this demo was to showcase that Navbar can create a more efficient workflow, thanks to the more simple and straightforward markup, a light footprint and powerful SCSS mixins that enable real flexible design.
  • We've used the same default responsive breakpoint of 768px, and this is another cool thing, you set one value in your SCSS variables and your JavaScript options exactly for your need, and without ALL the other breakpoints that Bootstrap Navbar comes with.

Bootstrap Extended SCSS

In terms of SCSS, this demo is very much the same with the default layout, we only removed a series of style rules with a couple of specific SCSS variable values, just because we can.

// navbar-bootstrap-extended.scss - mixins
@import "variables", "mixins", "root";

@include navbar-mobile-layout;
@include navbar-desktop-layout;
@include navbar-style;
@include nav-layout;
@include nav-style;
@include subnav-style;
@include main-menu;

As you can see, so far bootstrap-extended.scss is identical with navbar.scss, however for this demo we've setup a set of custom CSS variables:

// bootstrap-extended.html - custom CSS variable values
.navbar {
  --n-font-size: inherit;
  --n-bg: var(--bs-gray-dark);
  --n-subnav-bg: var(--bs-gray-200);
  --n-padding-x: 0.5rem;
  --n-padding-y: 0.5rem;
  --n-subnav-padding-x: .5rem;
  --n-subnav-padding-y: .5rem;
  --n-item-padding-y: .5rem;
  --n-item-padding-x: 1rem;
  --n-item-padding: var(--n-item-padding-y) var(--n-item-padding-x);
  --n-item-hover-bg: rgba(255,255,255,.15);
  --n-item-hover-color: rgba(255,255,255,.65);
  --n-item-active-bg: var(--bs-primary);
  --n-item-active-color: var(--bs-white);
  --n-subnav-item-hover-color: rgba(0,0,0,.75);
  --n-subnav-item-hover-bg: rgba(0,0,0,.125);
  --n-subnav-item-active-color: var(--bs-white);
  --n-subnav-item-active-bg: var(--bs-primary);
  --n-subnav-larger-width: calc(var(--n-subnav-width) * 3 + var(--n-subnav-padding-x) * 2);
}
@media (min-width: 768px) {
  .navbar {
    --n-padding-y: 0;
    --n-item-padding-y: 1rem;
    --n-item-padding-x: 1rem;
  }
}
  • since this markup is our Navbar markup we can flex some variables to customize the layout, mainly the spacing;
  • the rest of it is for the removal of background colors, for all active, hover and default state.
